🤔 Що таке парадигма програмування?

Парадигма — це стиль мислення, спосіб підходу до розв'язання задач. Уявіть, що ви готуєте борщ. Один кухар діє "крок за кроком", інший — "збирає інгредієнти як об'єкти", третій просто "описує, який борщ хоче".

⚡ Імперативна парадигма

Ідея: Даємо комп'ютеру чіткі інструкції — крок за кроком, що робити. Як рецепт приготування їжі або інструкція до меблів IKEA.

Приклад з ігор

Комп'ютерна гра:
1. Якщо натиснуто "W" — рухатись вперед
2. Якщо натиснуто "Пробіл" — стрибок
3. Якщо зіткнення з ворогом — відняти HP
4. Якщо HP = 0 — Game Over
Мови: C, Pascal, Python (базовий), Assembly

🎯 Об'єктно-орієнтована парадигма (ООП)

Ідея: Усе в світі — це об'єкти з властивостями та поведінкою. Як у грі Sims: кожен персонаж має характеристики та може виконувати дії.

Життєвий приклад: Автомобіль

Об'єкт "Авто":
• Властивості: колір, швидкість, рівень палива
• Методи: їхати(), гальмувати(), заправити()
• Стан: заведений/заглушений
🏦

Банківський рахунок

Об'єкт "Рахунок":
• Властивості: номер, баланс, валюта
• Методи: зняти(), покласти(), перевести()
• Інкапсуляція: баланс захищений паролем!

📋 Декларативна парадигма

Ідея: Описуємо ЩО хочемо отримати, а не ЯК це зробити. Як замовлення в ресторані: "Стейк середньої прожарки" — не кажемо шефу, як смажити!

🍽️ Конструктор замовлення

Опишіть, що ви хочете отримати (декларативно):

/* SQL: Декларативний запит */ SELECT назва, ціна FROM меню WHERE тип = 'піца' AND веган = true AND гострота < 3 ORDER BY ціна ASC;

💡 Ми не кажемо СУБД ЯК шукати — просто описуємо бажаний результат!

🐙 GitHub — командна робота

Уявіть, що ви пишете роман удвох з другом. Без GitHub: надсилаєте файли "роман_фінал_2_справжній_фінал.docx". З GitHub: бачите всі зміни, хто що написав, можете повернутися до будь-якої версії.
